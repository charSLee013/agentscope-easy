# Implement

## 执行守则

- 以 `Plan.md` 为执行真相来源。
- 一次只推进一个里程碑。
- diff 保持聚焦，不在未更新计划的情况下扩 scope。
- 每个里程碑结束后必须执行验证。
- 失败后先修复，再继续。
- 必需 artifact 一旦存在，只允许原位更新，不允许在收尾阶段删除重建。
- 持续更新 `Documentation.md`。

## 启动前检查

- 读取 `Prompt.md`
- 读取 `Plan.md`
- 读取 `Documentation.md`
- 确认当前里程碑、已知阻塞与必需 artifact 清单

## 里程碑循环

1. 明确当前里程碑目标。
2. 做最小实现或最小研究推进。
3. 执行该里程碑的验证命令。
4. 若失败，立即修复并重试。
5. 更新 `Documentation.md` 的状态、决策和审计日志。
6. 完成后再进入下一个里程碑。

## 子代理与搜索

- 允许使用本地 subagents 做边界清晰的探索、验证与风险扫描。
- 允许在事实可能变化时进行 web search，但优先使用一手来源。
- 所有重要发现都要回写到工作文档。

## 文档更新规则

- `Prompt.md`：仅在规格变化时更新。
- `Plan.md`：仅在里程碑、架构意图、验收标准或风险变化时更新。
- `Documentation.md`：每个里程碑循环都更新；出现失败或漂移时立即更新。

## 最终收口

1. 停止新增 scope。
2. 将 `Documentation.md` 切到真实最终状态，并填写 `## 最终验证证据`。
3. 运行 `finalize_long_horizon_run.py`。
4. 若 finalizer 失败，记录失败并停止，不继续盲修。

## 禁止事项

- 不跳过验证。
- 不在验证失败时继续推进后续工作。
- 不让 `Documentation.md` 落后于真实状态。
- 不在收尾阶段删除并重建 `Documentation.md` 或主交付物。
- 不在 active run 中提前把关键文件设为只读。
